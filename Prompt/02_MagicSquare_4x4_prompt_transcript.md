- 목적: Magic Square 4x4 TDD 설계 문서 작성을 위한 프롬프트 대화 이력 Export
- 범위: 현재 세션에서 오간 사용자 프롬프트 및 실행용 프롬프트 초안
- 형식: 대화형(role 기반) transcript

## Turn 1
### User
```text
너는 Cursor.AI용 프롬프트 설계자다.
나는 Magic Square 4x4 TDD 설계 문서를 만들기 위한 프롬프트가 필요하다.
지금 설계 문서를 만들지 말고,그 설계 문서를 만들 수 있는 실행용 프롬프트만 작성하라.
```

### Assistant (실행용 프롬프트 초안)
```text
너는 시니어 소프트웨어 아키텍트이자 TDD 코치다.
목표는 "Magic Square 4x4" 문제를 해결하기 위한 **TDD 설계 문서**를 작성하는 것이다.

[중요 지시]
- 지금 단계에서 코드 구현은 하지 말고, 설계 문서만 작성한다.
- 문서는 재현 가능하고 테스트 중심이어야 한다.
- 불확실한 요구사항은 반드시 "가정(Assumptions)" 섹션에 명시한다.
- 설명은 한국어로 작성한다.
- 출력은 반드시 아래 "출력 형식"을 정확히 따른다.

[문제 맥락]
- 대상: 4x4 마방진(Magic Square)
- 핵심 조건:
  1) 1~16의 정수를 중복 없이 정확히 1번씩 사용
  2) 각 행/열/주대각선 2개의 합이 모두 동일
- TDD 관점에서 문제를 분석하고, 테스트 전략과 설계를 문서화한다.

[작업 요구사항]
아래 항목을 순서대로 작성하라.

1. 문제 정의
- 문제 목표를 한 문단으로 명확히 요약
- 입력/출력/제약사항 정의
- 성공 조건(Definition of Done) 명시

2. 도메인 모델 및 용어
- 핵심 용어 정의(셀, 보드, 후보군, 제약 등)
- 데이터 표현 방식(예: 4x4 배열, 좌표 체계, 상태 스냅샷)

3. TDD 전략(핵심)
- Red-Green-Refactor 사이클을 이 문제에 어떻게 적용할지 설명
- 최소 기능부터 점진 확장하는 테스트 진화 계획 제시
- 테스트 피라미드/레이어(Unit 중심, 필요 시 Integration 보조) 명시

4. 테스트 설계
- 테스트 범주를 표로 제시:
  - 유효성 검증 테스트
  - 합 제약 테스트(행/열/대각선)
  - 중복/범위 제약 테스트
  - 생성/탐색 로직 테스트
  - 실패/예외/경계값 테스트
- 각 테스트에 대해 Given-When-Then 형식으로 대표 케이스 작성
- 우선순위(P0/P1/P2)와 목적을 함께 명시

5. 기능 분해 및 책임 할당
- 모듈/클래스/함수 수준의 책임 분리 제안
- 각 컴포넌트의 입력/출력/부작용 명시
- 테스트 가능성을 높이기 위한 설계 원칙(순수 함수, 의존성 분리 등) 명시

6. 알고리즘/탐색 전략(설계 수준)
- 가능한 접근법(백트래킹, 제약 전파 등) 비교
- 선택한 접근법의 이유(복잡도, 구현 난이도, 테스트 용이성)
- 성능 고려사항 및 최적화 포인트(가지치기 규칙) 제시

7. 단계별 구현 계획(TDD 기반)
- Step 1..N 형태로 작성
- 각 단계마다:
  - 먼저 작성할 실패 테스트
  - 통과를 위한 최소 구현 아이디어
  - 리팩터링 포인트
- 각 단계의 완료 조건을 체크리스트로 제시

8. 리스크 및 대응 전략
- 발생 가능한 기술 리스크와 완화 방안
- 테스트 취약 지점과 보강 전략

9. 최종 검증 계획
- 정확성 검증 방식
- 회귀 테스트 전략
- 문서/코드 일치성 점검 항목

[출력 형식]
- 반드시 아래 헤더 구조를 그대로 사용한다.
# Magic Square 4x4 TDD 설계 문서

## 1. 문제 정의

## 4. 테스트 설계

## 6. 알고리즘/탐색 전략
## 7. 단계별 구현 계획
## 8. 리스크 및 대응 전략
## 9. 최종 검증 계획
## 부록 A. 테스트 케이스 인덱스
## 부록 B. 가정(Assumptions)

[품질 기준]
- 모호한 표현 금지(“적당히”, “필요 시” 최소화)
- 각 섹션은 실행 가능한 수준의 구체성 확보
- 테스트 항목은 누락 없이 추적 가능해야 함(요구사항 ↔ 테스트 매핑)
- 문서 길이는 충분히 상세하게 작성하되 불필요한 장황함은 피할 것
```
